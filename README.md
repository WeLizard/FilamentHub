# FilamentHub

FilamentHub - это веб-приложение для управления коллекцией филаментов для 3D-печати.

## Быстрый старт (с использованием Docker)

Это рекомендуемый способ для запуска приложения. Он автоматически настраивает и запускает все необходимые сервисы (веб-сервер, API, базу данных) в изолированных контейнерах.

### Предварительные требования

*   **Docker:** Убедитесь, что на вашем компьютере установлен [Docker Desktop](https://www.docker.com/products/docker-desktop/). Он включает в себя `docker` и `docker-compose`.

### Запуск приложения

1.  **Склонируйте репозиторий:**
    ```bash
    git clone https://github.com/your-username/filamenthub.git
    cd filamenthub
    ```

2.  **Создайте файл `.env`:**
    Скопируйте файл `.env.template` в новый файл с именем `.env`.
    ```bash
    cp .env.template .env
    ```
    В файле `.env` вы можете изменить настройки, например, пароли. Для быстрого старта можно оставить значения по умолчанию. **Важно:** Обязательно сгенерируйте новый `SECRET_KEY`.

3.  **Запустите приложение с помощью Docker Compose:**
    Откройте терминал в корневой папке проекта и выполните следующую команду:
    ```bash
    docker-compose up --build -d
    ```
    Эта команда соберет Docker-образы (если они еще не собраны), запустит все сервисы в фоновом режиме (`-d`) и применит миграции базы данных.

4.  **Приложение готово!**
    *   Веб-интерфейс будет доступен по адресу: [http://localhost:80](http://localhost:80) (если порт 80 свободен).
    *   API бэкенда будет доступно по адресу: [http://localhost:8000](http://localhost:8000).

### Остановка приложения

Чтобы остановить все запущенные сервисы, выполните:
```bash
docker-compose down
```

### Устранение неполадок

**Ошибка `column ... does not exist`**

Если вы видите в логах ошибки, связанные с отсутствием колонок в базе данных (например, `sqlalchemy.exc.ProgrammingError: ... column "visual_settings" of relation "filaments" does not exist`), это означает, что схема вашей базы данных устарела.

Это может произойти, если вы обновляете код приложения, но используете старый том базы данных от предыдущей версии.

**Решение:**

Самый простой способ это исправить — полностью пересоздать базу данных. **Внимание: это приведет к удалению всех данных в базе.**

1.  **Остановите и удалите контейнеры и тома:**
    ```bash
    docker-compose down -v
    ```
    Флаг `-v` удаляет тома, включая том с базой данных.

2.  **Запустите приложение снова:**
    ```bash
    docker-compose up --build -d
    ```
    Docker создаст новую, чистую базу данных и применит все последние миграции.

### Дополнительные команды

*   **Просмотр логов:**
    ```bash
    # Показать логи всех сервисов
    docker-compose logs -f

    # Показать логи конкретного сервиса (например, бэкенда)
    docker-compose logs -f backend
    ```
*   **Выполнение команд внутри контейнера:**
    ```bash
    # Открыть bash-сессию в контейнере бэкенда
    docker-compose exec backend bash
    ```
