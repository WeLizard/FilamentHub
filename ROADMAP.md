# FilamentHub - Roadmap развития проекта

## 🎯 Видение проекта
**FilamentHub** — централизованная платформа для управления материалами и настройками в 3D-печати, объединяющая производителей пластиков, пользователей и слайсеры (OrcaSlicer, SuperSlicer, Bambu Studio).

**Слоган:** "Ваш пластик уже настроен. Осталось только печатать."

---

## 📊 Текущее состояние (Baseline)

### Существующие компоненты:
- ✅ **3dcalc** - PHP-приложение с:
  - Калькулятором стоимости печати
  - API для управления материалами
  - G-code парсером (Cura, PrusaSlicer, SuperSlicer, OrcaSlicer, Bambu Studio, Creality)
  - Базовым UI (HTML/JS/TailwindCSS)
  - MySQL базой данных

### Референсные проекты:
- 📚 **Spoolman** - Python FastAPI + React (для изучения архитектуры API и БД)
- 📚 **OrcaSlicer** - C++ слайсер (для понимания интеграции)
- 📚 **spoolman2slicer** - Python скрипт для экспорта данных в слайсеры

#### 1.2 Модели данных и миграция из 3dcalc
**Сроки:** 2 недели

**Модели данных:**

```python
# Основные сущности
- Brand (Производитель)
  - id, name, description, website, logo_url, verified
  - created_at, updated_at
  
- Filament (Материал)
  - id, brand_id, name, material_type, color_hex
  - diameter (1.75/2.85), density, weight, price
  - settings (температуры, скорости - JSON)
  - created_at, updated_at
  
- Printer (Принтер)
  - id, manufacturer, model, type (FFF/SLA)
  
- Preset (Пресет настроек)
  - id, filament_id, printer_id, user_id
  - settings (JSON с параметрами)
  - rating, success_rate, print_count
  - is_official (от производителя)
  - created_at, updated_at
  
- User (Пользователь)
  - id, email, username, role (user/brand/admin)
  - api_key (для плагинов)
  
- Review (Отзыв)
  - id, preset_id, user_id, rating, comment
  - success (boolean), photo_url
```

**Задачи:**
- [ ] Создать SQLAlchemy модели
- [ ] Написать миграции Alembic
- [ ] Создать скрипт миграции данных из MySQL (3dcalc) в PostgreSQL
- [ ] Протестировать миграцию данных

#### 1.3 REST API эндпоинты
**Сроки:** 2 недели

**API структура:**

```
/api/v1/
├── brands/
│   ├── GET /                    # Список брендов
│   ├── GET /{id}                # Детали бренда
│   ├── POST /                   # Создать бренд (auth)
│   └── GET /{id}/filaments      # Материалы бренда
│
├── filaments/
│   ├── GET /                    # Список материалов (фильтры)
│   ├── GET /{id}                # Детали материала
│   ├── POST /                   # Добавить материал (brand auth)
│   └── GET /{id}/presets        # Пресеты для материала
│
├── presets/
│   ├── GET /                    # Список пресетов
│   ├── GET /{id}                # Детали пресета
│   ├── POST /                   # Создать пресет (auth)
│   ├── PUT /{id}                # Обновить пресет
│   └── GET /recommend           # Рекомендованные пресеты
│
├── printers/
│   ├── GET /                    # Список принтеров
│   └── GET /{id}                # Детали принтера
│
├── reviews/
│   ├── GET /preset/{id}         # Отзывы на пресет
│   └── POST /                   # Добавить отзыв (auth)
│
├── calculator/
│   ├── POST /estimate           # Рассчитать стоимость
│   └── POST /parse-gcode        # Парсинг G-code
│
└── auth/
    ├── POST /register
    ├── POST /login
    └── POST /api-key              # Генерация API ключа для плагинов
```

## 🛠️ Технический стек (финальный)

### Backend
- **Language:** Python 3.11+
- **Framework:** FastAPI
- **ORM:** SQLAlchemy 2.0
- **Database:** PostgreSQL 15
- **Cache:** Redis 7
- **Task Queue:** Celery (для фоновых задач)
- **Testing:** pytest, pytest-asyncio

### Frontend
- **Language:** TypeScript 5
- **Framework:** React 18
- **Build:** Vite
- **UI:** shadcn/ui + TailwindCSS
- **State:** TanStack Query
- **Forms:** React Hook Form + Zod
- **i18n:** react-i18next

### Plugin
- **Language:** Python 3.8+ (совместимость с OrcaSlicer)
- **UI:** wxPython (если нужен GUI)
- **Config:** TOML/JSON

### DevOps
- **Container:** Docker + Docker Compose
- **CI/CD:** GitHub Actions
- **Monitoring:** Prometheus + Grafana
- **Logging:** Sentry
- **CDN:** Cloudflare
- **Hosting:** VPS (Hetzner/Timeweb) или Cloud (AWS/GCP)

## 🚨 Риски и митигация

### Технические риски
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Сложность интеграции с OrcaSlicer | Средняя | Высокое | Альтернативный CLI подход |
| Проблемы масштабирования БД | Низкая | Среднее | PostgreSQL + индексы + кэш |
| Безопасность API | Средняя | Высокое | Rate limiting, JWT, валидация |

### Бизнес риски
| Риск | Вероятность | Влияние | Митигация |
|------|-------------|---------|-----------|
| Низкая вовлеченность производителей | Средняя | Высокое | Прямой контакт, демо, ценность |
| Конкуренция со Spoolman | Низкая | Среднее | Совместимость, уникальные фичи |
| Низкое качество пресетов | Средняя | Среднее | Модерация, система рейтингов |
