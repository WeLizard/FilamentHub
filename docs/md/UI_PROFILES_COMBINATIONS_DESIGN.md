# UI/UX дизайн для согласования профилей печати

## 🎯 Проблема

У нас есть **3 типа сущностей**, которые нужно согласовать между собой:

1. **PrinterProfile** (Профиль принтера) - настройки принтера + сопло
2. **PrintProfile** (Профиль печати) - настройки слоёв, скорости, заполнения
3. **Filament** (Материал) - самостоятельный объект (например, "PLA Red")
   - **Preset** (Настройки филамента) - привязан к Filament (один Filament может иметь много Preset)

**Связи между ними:**
- `PrinterProfile.default_print_profile_slug` → `PrintProfile.slug`
- `PrintProfile.compatible_printers` → массив `Printer.slug` (через `print_profile_printers`)
- `PrintProfile.compatible_filaments` → массив `Filament.slug` (через `print_profile_filaments`, опционально)
- `Preset.filament_id` → `Filament.id` (один Filament → много Preset)
- `Preset` → `Printer` (через `preset_printers`, many-to-many)

**Важно:** Filament - это самостоятельный объект, не профиль! Preset - это настройки для Filament.

**Задача:** Как пользователь будет выбирать и согласовывать:
1. PrinterProfile (какой принтер + сопло)
2. PrintProfile (какой профиль печати)
3. Filament (какой материал) + Preset (какие настройки для этого материала)

на сайте?

## 💡 Варианты решения

### Вариант 1: Конструктор комбинаций (рекомендуется)

**Где показывать:** Отдельная страница или модалка "Конструктор профилей"

**Логика:**
1. Пользователь выбирает **PrinterProfile** (принтер + сопло)
2. Система показывает совместимые **PrintProfile** для этого принтера
3. Пользователь выбирает **PrintProfile**
4. Система показывает совместимые **Filament** для этой комбинации (через `print_profile_filaments`)
5. Пользователь выбирает **Filament**
6. Система показывает совместимые **Preset** для выбранного филамента (и принтера, если есть связь через `preset_printers`)
7. Пользователь выбирает **Preset** и сохраняет комбинацию

**UI структура:**
```
┌─────────────────────────────────────────┐
│  Конструктор профилей                   │
├─────────────────────────────────────────┤
│                                         │
│  Шаг 1: Выберите принтер               │
│  [Ender 3 Pro] [Prusa MK4] [Voron]     │
│                                         │
│  Шаг 2: Выберите профиль печати        │
│  [0.20mm Standard] [0.15mm Fine] ...   │
│                                         │
│  Шаг 3: Выберите материал (Filament)   │
│  [PLA Red] [PETG Black] [TPU] ...      │
│                                         │
│  Шаг 4: Выберите настройки (Preset)    │
│  [Официальный] [Взвешенный] [Пользовательский] │
│                                         │
│  [Сохранить комбинацию] [Экспорт]      │
└─────────────────────────────────────────┘
```

**Преимущества:**
- Понятная последовательность выбора
- Система автоматически фильтрует несовместимые варианты
- Можно сохранять готовые комбинации

**Недостатки:**
- 4 шага могут быть слишком много
- Нужно запоминать, что выбрано на каждом шаге

### Вариант 2: Обратный выбор (от филамента)

**Где показывать:** На странице филамента (`FilamentDetailPage`)

**Логика:**
1. Пользователь открывает страницу филамента (например, "PLA Red")
2. Система показывает **совместимые принтеры** (выведенные через Preset → Printer или PrintProfile → Printer)
3. Пользователь выбирает принтер (или PrinterProfile)
4. Система показывает **совместимые PrintProfile** для этого принтера
5. Пользователь выбирает PrintProfile
6. Система показывает **совместимые Preset** для выбранного филамента (и принтера, если есть связь)

**UI структура:**
```
Страница филамента (PLA)
├── Блок "Выберите принтер"
│   └── Список совместимых принтеров
│       └── При клике → показать PrintProfile для этого принтера
│
├── Блок "Рекомендуемые профили печати"
│   └── (показывается после выбора принтера)
│
└── Блок "Выберите пресет"
    └── (показывается после выбора PrintProfile)
```

**Преимущества:**
- Естественный флоу: "У меня есть PLA, что с ним можно делать?"
- Начинается с филамента (что обычно известно пользователю)

**Недостатки:**
- Не все пользователи знают, какой филамент у них есть
- Нужно делать несколько запросов к API

### Вариант 3: Матрица совместимости

**Где показывать:** В профиле пользователя (уже есть заглушка "Комбинации профилей")

**Логика:**
1. Пользователь видит таблицу/матрицу своих профилей
2. Система автоматически находит совместимые комбинации:
   - PrinterProfile → PrintProfile (через `default_print_profile_slug`)
   - PrintProfile → Filament (через `print_profile_filaments`)
   - Filament → Preset (один к одному)
3. Показываются готовые комбинации: "Ender 3 Pro + 0.20mm Standard + PLA"
4. Пользователь может сохранить комбинацию или экспортировать в OrcaSlicer

**UI структура:**
```
┌─────────────────────────────────────────┐
│  Комбинации профилей                    │

│                                         │
│  ✅ Ender 3 Pro 0.4mm                   │

│     [Экспорт] [Сохранить]               │

│  ✅ Voron 2.4 350                       │
│     → 0.15mm Fine                       │
│     → PETG Black (eSUN)                 │

│     [Экспорт] [Сохранить]               │
│                                         │

│                                         │
```

**Преимущества:**
- Видно все свои профили сразу
- Автоматически находятся совместимые комбинации
- Можно видеть, чего не хватает (принтер без профиля печати)

**Недостатки:**
- Может быть сложно для новичков
- Нужно много данных для отображения

### Вариант 4: Умный выбор (рекомендации)

**Где показывать:** В разных местах, в зависимости от контекста

**Логика:**
1. **Если пользователь выбрал принтер** → показать рекомендуемые PrintProfile + Filament + Preset
2. **Если пользователь выбрал филамент** → показать рекомендуемые принтеры + PrintProfile + Preset
3. **Если пользователь выбрал PrintProfile** → показать совместимые принтеры + филаменты + пресеты

**UI структура:**
```
На странице принтера (Ender 3 Pro):
┌─────────────────────────────────────────┐
│  Рекомендуемые комбинации               │
├─────────────────────────────────────────┤
│  ✅ 0.20mm Standard + PLA               │
│     [Пресет: Официальный]               │
│     [Использовать эту комбинацию]       │
│                                         │
│  ✅ 0.15mm Fine + PETG                  │
│     [Пресет: Взвешенный]                │
│     [Использовать эту комбинацию]       │
└─────────────────────────────────────────┘
```

**Преимущества:**
- Умные рекомендации на основе данных
- Контекстный выбор в зависимости от того, что пользователь уже выбрал
- Минимум шагов для пользователя

**Недостатки:**
- Нужен хороший алгоритм рекомендаций
- Может быть неочевидно, откуда берутся рекомендации

## 🔄 Алгоритм автоматического поиска комбинаций

### Шаг 1: Собираем профили пользователя

```typescript
const userPrinterProfiles = await getPrinterProfiles(userId);
const userPrintProfiles = await getPrintProfiles(userId);
const userPresets = await getPresets(userId); // через Filament
```

### Шаг 2: Находим связи

```typescript
// 1. PrinterProfile → PrintProfile (через default_print_profile_slug)
for (const pp of userPrinterProfiles) {
  if (pp.default_print_profile_slug) {
    const printProfile = findPrintProfileBySlug(pp.default_print_profile_slug);
    if (printProfile && printProfile.owner_user_id === userId) {
      // Найдена связь: pp → printProfile
    }
  }
}

// 2. PrintProfile → Filament (через print_profile_filaments)
for (const printProfile of userPrintProfiles) {
  const filaments = await getFilamentsByPrintProfile(printProfile.id);
  for (const filament of filaments) {
    // 3. Filament → Preset (один Filament может иметь много Preset)
    const presets = await getPresetsByFilament(filament.id);
    for (const preset of presets) {
      // Проверяем, есть ли связь Preset → Printer (через preset_printers)
      // и совпадает ли с принтером из PrinterProfile
      if (preset.user_id === userId || preset.is_official) {
        // Найдена полная комбинация!
      }
    }
  }
}
```

1. `GET /api/v1/profiles/combinations` - получить комбинации профилей пользователя
