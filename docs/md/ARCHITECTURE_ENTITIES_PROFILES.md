# Архитектура: Разделение на сущности и профили

## Концепция

В FilamentHub используется архитектура с разделением на **базовые сущности** (Entities) и **профили** (Profiles):

- **Базовые сущности** - описание объекта (материал, принтер, бренд)
- **Профили** - конкретные настройки/конфигурации для объекта

## Структура данных

### 1. Филаменты

#### `Filament` (базовая сущность)
**Что хранит:**
- Базовую информацию о материале
- Производитель (brand)
- Название, тип материала (PLA, PETG, ABS)
- Цвет, визуальные характеристики
- Физические свойства (диаметр, плотность)
- Цену, вес катушки

**Пример:**
```
Filament {
  id: 1
  brand_id: 5
  name: "PLA Red"
  material_type: "PLA"
  color_hex: "#FF0000"
  diameter: 1.75
  ...
}
```

**Ключевые поля:**
- `id` - уникальный идентификатор
- `brand_id` - связь с производителем
- `slug` - URL-friendly идентификатор
- `active` - активен ли в каталоге

#### `Preset` (профиль настроек)
**Что хранит:**
- Конкретные настройки печати для материала
- Температуры (extruder, bed)
- Скорости (print, travel)
- Дополнительные параметры (layer_height, flow_rate, retraction)
- Связь с принтерами через `preset_printers`

**Пример:**
```
Preset {
  id: 10
  filament_id: 1  # Связь с Filament
  name: "Официальный пресет Bestfilament"
  extruder_temp: 200
  bed_temp: 60
  print_speed: 50
  ...
}
```

**Ключевые поля:**
- `id` - уникальный идентификатор
- `filament_id` - **связь с базовым филаментом**
- `user_id` - кто создал
- `is_official` - официальный от производителя или пользовательский
- `orcaslicer_settings` - расширенные настройки OrcaSlicer

**Связи:**
- `Filament` (1:many) - один филамент может иметь множество пресетов
- `PresetPrinter` (many:many) - пресет может быть совместим с несколькими принтерами

### 2. Принтеры

#### `Printer` (базовая сущность)
**Что хранит:**
- Базовую информацию о принтере
- Производитель, модель
- Технические характеристики (размеры стола, высота)
- Ограничения (максимальные температуры)
- Варианты сопел (nozzle_options)

**Пример:**
```
Printer {
  id: 1
  name: "Voron 2.4"
  manufacturer: "Voron Design"
  model: "2.4"
  build_volume_x: 350
  build_volume_y: 350
  build_volume_z: 280
  nozzle_options: [0.4, 0.6, 0.8]
  ...
}
```

**Ключевые поля:**
- `id` - уникальный идентификатор
- `slug` - URL-friendly идентификатор
- `model_id` - внутренний ID модели от производителя
- `active` - активен ли в каталоге

#### `PrinterProfile` (профиль настроек)
**Что хранит:**
- Конкретные настройки принтера для пользователя
- Размеры печатной области (может отличаться от базовых)
- Конкретные сопла (nozzle_diameters: [0.4, 0.6])
- G-code (start_gcode, end_gcode)
- Полные настройки OrcaSlicer

**Пример:**
```
PrinterProfile {
  id: 5
  printer_id: 1  # Связь с Printer
  owner_user_id: 10
  name: "Мой Ворон 2.4 350"
  nozzle_diameters: [0.4, 0.6]
  printable_area: {"x": 350, "y": 350}
  printable_height_mm: 280
  ...
}
```

**Ключевые поля:**
- `id` - уникальный идентификатор
- `printer_id` - **связь с базовым принтером**
- `owner_user_id` - владелец профиля
- `nozzle_diameters` - список сопел для этого профиля
- `orcaslicer_settings` - полные настройки OrcaSlicer

**Связи:**
- `Printer` (many:1) - несколько профилей могут ссылаться на один принтер
- Пользователь может иметь несколько профилей для одного принтера (разные конфигурации)

### 3. Профили печати

#### `PrintProfile` (профиль настроек печати)
**Что хранит:**
- Настройки печати (не привязан к конкретному принтеру или филаменту напрямую)
- Толщину слоя (layer_height_mm)
- Категорию (quality, speed, special)
- Совместимость с принтерами (через `compatible_printers` - список slugs)
- Совместимость с филаментами (через `compatible_filaments` - список slugs)

**Пример:**
```
PrintProfile {
  id: 20
  owner_user_id: 10
  name: "0.2mm Standard"
  category: "quality"
  quality_tier: "standard"
  layer_height_mm: 0.2
  compatible_printers: ["voron-2.4", "prusa-mk3"]
  compatible_filaments: ["pla", "petg"]
  ...
}
```

**Ключевые поля:**
- `id` - уникальный идентификатор
- `owner_user_id` - владелец (None для системных)
- `compatible_printers` - список slugs принтеров (JSON)
- `compatible_filaments` - список slugs филаментов (JSON)
- `is_official` - системный или пользовательский

**Связи:**
- Связь с принтерами через `compatible_printers` (список slugs)
- Связь с филаментами через `compatible_filaments` (список slugs)
- `PrintProfilePrinter` (many:many) - для более сложных связей
- `PrintProfileFilament` (many:many) - для более сложных связей

## Принципы использования

### 1. Разделение ответственности

**Базовые сущности (`Filament`, `Printer`):**
- Описывают объект в общем виде
- Хранят информацию, которая не меняется
- Один раз создаются в системе
- Используются в каталоге, поиске, фильтрах

**Профили (`Preset`, `PrinterProfile`, `PrintProfile`):**
- Содержат конкретные настройки/конфигурации
- Могут быть множественными для одной сущности
- Создаются пользователями или импортируются
- Используются для настройки печати

### 2. Принцип не дублирования

**НЕ создавать:**
- ❌ Отдельный `Filament` для каждого цвета (использовать `color_hex` в одном `Filament`)
- ❌ Отдельный `Printer` для каждого сопла (использовать `nozzle_diameters` в `PrinterProfile`)
- ❌ Отдельный `Preset` для одинаковых настроек (использовать один пресет)

**Создавать:**
- ✅ Отдельный `Preset` для разных настроек печати одного материала
- ✅ Отдельный `PrinterProfile` для разных конфигураций одного принтера
- ✅ Отдельный `PrintProfile` для разных режимов печати

### 3. Сопоставление при синхронизации

#### При синхронизации филаментов:
1. Ищем существующий `Filament` по имени + material_type (во всех брендах)
2. Если найден - используем его, создаём новый `Preset`
3. Если не найден - создаём `Filament` как черновик в "User Materials"

#### При синхронизации принтеров:
1. Ищем существующий `Printer` по manufacturer + model + model_id
2. Если найден - используем его, создаём/обновляем `PrinterProfile`
3. Если не найден - создаём новый `Printer`
4. **Важно:** Не создавать отдельный `PrinterProfile` для каждого сопла - объединять в один

#### При синхронизации профилей печати:
1. Для системных - ищем по `layer_height_mm` + `quality_tier` + `source="orca_system"`
2. Для пользовательских - ищем по `slug` + `owner_user_id`
3. Если найден - обновляем
4. Если не найден - создаём новый

### 4. Использование в UI

**Каталог:**
- Показываем `Filament` (базовые сущности)
- При клике - показываем список `Preset` для этого филамента

**Настройка печати:**
1. Пользователь выбирает `Filament` → показываем список `Preset`
2. Пользователь выбирает `Printer` → показываем список `PrinterProfile`
3. Пользователь выбирает `PrintProfile` → показываем совместимые комбинации

**Персонализация:**
- Определяем принтеры пользователя через `PrinterProfile.owner_user_id`
- Показываем только релевантные `Filament` и `Preset` для его принтеров

## Примеры использования

### Пример 1: Один материал, несколько пресетов

```
Filament: "PLA Red" (id=1)
  ├─ Preset: "Официальный пресет" (extruder=200, bed=60)
  ├─ Preset: "Для быстрой печати" (extruder=210, bed=60, speed=80)
  └─ Preset: "Для качественной печати" (extruder=195, bed=60, speed=30)
```

### Пример 2: Один принтер, несколько профилей

```
Printer: "Voron 2.4" (id=1)
  ├─ PrinterProfile: "Мой Ворон 350" (nozzle_diameters=[0.4, 0.6], height=280)
  └─ PrinterProfile: "Ворон для вазочек" (nozzle_diameters=[0.8], height=350)
```

### Пример 3: Профили печати с совместимостью

```
PrintProfile: "0.2mm Standard" (id=10)
  ├─ compatible_printers: ["voron-2.4", "prusa-mk3"]
  └─ compatible_filaments: ["pla", "petg"]

PrintProfile: "Vase Mode" (id=11)
  ├─ compatible_printers: ["voron-2.4"]  # Только для принтеров с большим столом
  └─ compatible_filaments: ["pla", "petg", "abs"]
```

## Ключевые принципы

1. ✅ **Базовые сущности** - уникальные объекты в системе
2. ✅ **Профили** - множественные настройки для сущностей
3. ✅ **Не дублировать** базовые сущности без необходимости
4. ✅ **Объединять** вариативные параметры (сопла) в профилях
5. ✅ **Сопоставлять** существующие сущности при синхронизации
6. ✅ **Использовать** профили для персонализации

## Связи между сущностями

```
Brand (1) ───< (many) Filament (1) ───< (many) Preset
                                                      │
                                                      └───> (many:many) Printer

Printer (1) ───< (many) PrinterProfile

PrintProfile ───> (many) Printer (через compatible_printers)
             └───> (many) Filament (через compatible_filaments)
```
