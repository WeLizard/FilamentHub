# Параметры филамента для агрегации производителей

## Анализ: Что важно для производителей филаментов

На основе исследования того, что обычно указывают производители в даташитах и спецификациях.

## Ключевое разделение

### Универсальные параметры материала (НЕ зависят от принтера)
**Эти параметры применимы к любому принтеру и важны для агрегации:**

### 1. Физические свойства материала
- **Тип материала** (`filament_type`) - PLA, ABS, PETG, TPU, etc.
- **Диаметр** (`filament_diameter`) - 1.75 мм или 2.85 мм (универсально)
- **Плотность** (`filament_density`) - г/см³ (для расчета веса, универсально)
- **Цвет** (`default_filament_colour`) - визуальное свойство
- **Производитель** (`filament_vendor`) - кто производит

**Все производители указывают это в спецификациях!**

### 2. Температурные характеристики (рекомендуемые производителем)
**Это то, что производители указывают в даташитах:**

- **Диапазон температуры экструдера:**
  - `nozzle_temperature_range_low` - минимальная рекомендованная
  - `nozzle_temperature_range_high` - максимальная рекомендованная
  - `nozzle_temperature` - оптимальная (средняя из диапазона)

- **Температура стола:**
  - `hot_plate_temp` или `cool_plate_temp` - рекомендованная температура стола
  - Производители обычно указывают диапазон (например 50-70°C для PLA)

**Почему важно:** Это единственные температурные параметры, которые производитель указывает в спецификациях. Они универсальны и не зависят от принтера.

### 3. Рекомендуемые настройки печати (базовые)
**То, что некоторые производители указывают на упаковке или сайте:**

- **Максимальная объемная скорость** (`filament_max_volumetric_speed`) - максимальный объем филамента в секунду
- **Коэффициент потока** (`filament_flow_ratio`) - иногда указывается производителем для особых составов

**Но не все производители это указывают!**

### 4. Усадка материала
**Универсальное свойство материала:**

- `filament_shrink` - усадка филамента (например "99.8%")
- `temperature_vitrification` - температура витрификации

**Производители обычно не указывают, но это свойство материала, не принтера.**

## Параметры, которые НЕ универсальны (зависят от принтера)

### ❌ Ретракция (Retraction)
**Зависит от типа экструдера:**
- Direct Drive: 0.5-3 мм
- Bowden: 4-8 мм
- Скорость ретракции тоже зависит от принтера

**Вывод:** Не нужно для производителей, но нужно для пользовательских пресетов (после тестирования на конкретном принтере)

### ❌ Вентиляторы (Fans)
**Зависит от принтера:**
- Количество вентиляторов
- Расположение
- Производительность

**Вывод:** Не нужно для производителей, нужно для пользовательских профилей

### ❌ Скорость печати
**Зависит от принтера:**
- Жесткость рамки
- Кинематика
- Мощность двигателей

**Вывод:** Это параметр process, не filament. Производители не указывают.

### ❌ Pressure Advance
**Зависит от принтера:**
- Жесткость экструдера
- Длина пути филамента
- Тип экструдера

**Вывод:** Не универсально, не нужно для производителей

### ❌ G-code скрипты
**Зависит от прошивки и принтера:**
- Команды специфичны для Marlin/Klipper
- Зависит от конкретной модели принтера

**Вывод:** Не нужно для производителей

### ❌ Камера и фильтрация
**Зависит от принтера:**
- Есть ли закрытая камера
- Есть ли система фильтрации

**Вывод:** Не нужно для производителей

### ❌ Мультитул параметры
**Зависит от принтера:**
- Есть ли мультитул
- Какая система смены инструмента

**Вывод:** Не нужно для производителей

## Что нужно для агрегации производителей

### ОБЯЗАТЕЛЬНО (уровень 1):

1. **Физические свойства:**
   - `filament_type` - тип материала (PLA, ABS, etc.)
   - `filament_vendor` - производитель
   - `filament_diameter` - диаметр (1.75 или 2.85)
   - `filament_density` - плотность
   - `default_filament_colour` - цвет

2. **Температурные диапазоны:**
   - `nozzle_temperature_range_low` - мин. температура
   - `nozzle_temperature_range_high` - макс. температура
   - `nozzle_temperature` - оптимальная (можно вычислить как среднее)
   - `hot_plate_temp` / `cool_plate_temp` - температура стола

3. **Стоимость:**
   - `filament_cost` - цена за грамм (для калькулятора)

**Итого: ~10-12 параметров для базового профиля производителя**

### ЖЕЛАТЕЛЬНО (уровень 2):

4. **Характеристики печати:**
   - `filament_max_volumetric_speed` - максимальная объемная скорость
   - `filament_flow_ratio` - коэффициент потока (если указан производителем)

5. **Усадка:**
   - `filament_shrink` - усадка материала
   - `temperature_vitrification` - температура витрификации

**Итого: ~15-17 параметров для продвинутого профиля производителя**

---

## Что НЕ нужно для производителей

### Параметры, которые заполняются пользователями после тестирования:

- ❌ Ретракция (зависит от экструдера)
- ❌ Вентиляторы (зависит от принтера)
- ❌ Pressure Advance (зависит от принтера)
- ❌ G-code скрипты (зависит от принтера)
- ❌ Камера и фильтрация (зависит от принтера)
- ❌ Мультитул параметры (зависит от принтера)
- ❌ Скорости печати (это process параметр)
- ❌ Совместимость принтеров

**Эти параметры появляются в пользовательских пресетах после тестирования на конкретном принтере!**

## Архитектура для FilamentHub

### Модель Filament (уже есть):
- `material_type` → `filament_type`
- `diameter` → `filament_diameter`
- `density` → `filament_density`
- `color_hex` → `default_filament_colour`
- `price_per_kg` → `filament_cost`
- `brand.name` → `filament_vendor`

### Модель Preset (базовые настройки производителя):

**Для официальных пресетов (от производителей):**

1. **Температуры:**
   - `extruder_temp` → `nozzle_temperature` (оптимальная)
   - `bed_temp` → `hot_plate_temp` / `cool_plate_temp`
   - ➕ Добавить: `extruder_temp_min`, `extruder_temp_max` → диапазон температур

2. **Расширенные параметры в JSON:**
   - `nozzle_temperature_range_low/high` - диапазон температур
   - `filament_max_volumetric_speed` - макс. объемная скорость
   - `filament_flow_ratio` - коэффициент потока (если указан)
   - `filament_shrink` - усадка
   - `temperature_vitrification` - температура витрификации

**Для пользовательских пресетов:**

3. **Дополнительно (после тестирования на принтере):**
   - Ретракция (retraction_length, retraction_speed)
   - Вентиляторы (fan_min_speed, fan_max_speed)
   - Pressure Advance (pressure_advance)
   - И другие принтер-специфичные параметры

## Рекомендация для FilamentHub (РЕАЛИЗОВАНО)

### Подход:

**1. Модель Preset:**
- **Базовые поля** (SQL колонки): температуры, ретракция, вентиляторы, flow rate и т.д.
- **JSON поле `orcaslicer_settings`**: все расширенные параметры OrcaSlicer
- **Гибкость**: можно хранить сколько угодно параметров в JSON

**2. Официальные пресеты (от производителей):**
- Базовые параметры в SQL колонках (температуры, ретракция, вентиляторы)
- Расширенные параметры в JSON (температурные диапазоны, volumetric speed и т.д.)
- ~12-15 базовых параметров + JSON для расширенных

**3. Пользовательские пресеты:**
- Все базовые параметры в SQL колонках
- Все расширенные параметры в JSON (pressure advance, G-code скрипты, мультитул параметры и т.д.)
- ~30-50+ параметров (включая принтер-специфичные)

**4. При экспорте в OrcaSlicer:**
- Используем базовые поля из SQL колонок
- Дополняем параметрами из `orcaslicer_settings` (JSON)
- Конвертируем все значения в массивы строк (формат OrcaSlicer)

**5. При импорте из OrcaSlicer:**
- Парсим JSON профиль
- Базовые параметры → SQL колонки (если есть соответствие)
- Все остальные параметры → JSON поле `orcaslicer_settings`
- Сохраняем всё для полной совместимости
