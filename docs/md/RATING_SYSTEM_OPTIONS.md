# Варианты реализации системы рейтингов и успешности

## Проблема

**Текущая ситуация:**
- FilamentReview не связан с Preset
- Preset.rating - статическое поле, не обновляется автоматически
- Невозможно узнать, какой пресет использовался при печати

**Требования:**
- У каждого пресета свой рейтинг и успешность печати
- У филамента общий рейтинг и успешность (на основе всех пресетов)
- Официальный пресет имеет свой рейтинг и успешность

---

## Вариант 1: Отзывы привязаны к пресетам (РЕКОМЕНДУЕТСЯ) ⭐

### Архитектура:

```
FilamentReview
├── filament_id (обязательно)
├── preset_id (обязательно) ← НОВОЕ ПОЛЕ
├── user_id
├── success (bool)
├── rating (1-5)
└── comment
```

### Логика:

**Рейтинг пресета:**
- `preset.rating` = средний рейтинг всех отзывов этого пресета
- `preset.success_rate` = процент успешных печатей по этому пресету
- Обновляется автоматически при создании/обновлении отзыва

**Рейтинг филамента:**
- `filament.rating` = средний рейтинг всех отзывов (по всем пресетам)
- `filament.success_rate` = процент успешных печатей по всем отзывам
- Или можно взвешенное среднее по пресетам (с учетом usage_count)

### Преимущества:
✅ Логично: отзыв о конкретном использовании пресета
✅ Точная статистика для каждого пресета
✅ Можно фильтровать отзывы по пресету
✅ Автоматическое обновление рейтингов пресетов

### Недостатки:
❌ Нужно добавлять поле preset_id в FilamentReview
❌ Нужна миграция БД
❌ При создании отзыва нужно обязательно указывать пресет

### Изменения в БД:
```sql
ALTER TABLE filament_reviews ADD COLUMN preset_id INTEGER REFERENCES presets(id);
CREATE INDEX ix_filament_reviews_preset_id ON filament_reviews(preset_id);
```

### Изменения в коде:
- Добавить `preset_id` в FilamentReview модель
- При создании отзыва обязательно указывать preset_id
- Добавить вычисление рейтинга пресета из отзывов
- Добавить `success_rate` в модель Preset (или вычислять динамически)

## Вариант 2: Отзывы о филаменте + отдельные рейтинги пресетов

### Архитектура:

```
FilamentReview
├── filament_id (обязательно)
├── preset_id (опционально) ← НОВОЕ ПОЛЕ
├── user_id
├── success (bool)
├── rating (1-5)
└── comment

Preset
├── rating (вычисляемое или ручное)
├── success_rate (вычисляемое или ручное)
└── usage_count
```

### Логика:

**Рейтинг пресета:**
- Если есть отзывы с `preset_id` → средний рейтинг этих отзывов
- Если нет отзывов → можно использовать `usage_count` как индикатор популярности
- Или пользователи могут ставить рейтинг пресету отдельно от отзыва о филаменте

**Рейтинг филамента:**
- Средний рейтинг всех отзывов (независимо от пресета)
- Успешность = процент успешных печатей по всем отзывам

### Преимущества:
✅ Гибкость: можно оставить отзыв о филаменте без указания пресета
✅ Можно добавлять рейтинг пресету отдельно (например, кнопка "Лайк пресету")
✅ Постепенная миграция: старые отзывы остаются валидными

### Недостатки:
❌ Менее точная статистика по пресетам (если preset_id опциональный)
❌ Нужна логика для двух сценариев (с пресетом и без)

## Вариант 3: Гибридный подход (отзывы + отдельные оценки пресетов)

### Архитектура:

```
FilamentReview
├── filament_id (обязательно)
├── preset_id (опционально)
├── user_id
├── success (bool)
├── rating (1-5) - рейтинг филамента
└── comment

PresetRating ← НОВАЯ ТАБЛИЦА
├── preset_id
├── user_id
├── rating (1-5) - рейтинг пресета
├── success (bool)
└── created_at
```

### Логика:

**Рейтинг пресета:**
- Средний рейтинг из таблицы PresetRating
- Успешность = процент успешных печатей из PresetRating

**Рейтинг филамента:**
- Средний рейтинг из FilamentReview
- Успешность = процент успешных печатей из FilamentReview

### Преимущества:
✅ Четкое разделение: отзыв о филаменте vs рейтинг пресета
✅ Можно оценить пресет отдельно от филамента
✅ Гибкость: можно оставить отзыв без оценки пресета

### Недостатки:
❌ Две системы рейтингов (сложнее)
❌ Нужна новая таблица PresetRating
❌ Больше полей для заполнения пользователем

## Вариант 4: Отзывы привязаны к пресетам + рейтинг филамента вычисляется

### Архитектура:

```
FilamentReview
├── filament_id (обязательно)
├── preset_id (обязательно) ← НОВОЕ ПОЛЕ
├── user_id
├── success (bool)
├── rating (1-5) - рейтинг опыта печати (филамент + пресет)
└── comment
```

### Логика:

**Рейтинг пресета:**
- Средний рейтинг всех отзывов этого пресета
- Успешность = процент успешных печатей по этому пресету

**Рейтинг филамента:**
- Взвешенное среднее рейтингов пресетов (с учетом usage_count)
- Или средний рейтинг всех отзывов (простое среднее)
- Успешность = процент успешных печатей по всем отзывам

### Преимущества:
✅ Один источник данных (отзывы)
✅ Автоматическое обновление всех рейтингов
✅ Точная статистика

### Недостатки:
❌ Нужно обязательно указывать пресет при создании отзыва

---

## Рекомендация: Вариант 1 ⭐

**Почему:**
1. Логично: пользователь печатает с конкретным пресетом, значит отзыв должен быть о конкретном пресете
2. Точность: можно точно оценить качество каждого пресета
3. Простота: одна система рейтингов, автоматическое обновление
4. Полезность: пользователи видят, какие пресеты работают хорошо, а какие нет

**Реализация:**
1. Добавить `preset_id` в FilamentReview (обязательное поле)
2. Добавить вычисляемые поля `rating` и `success_rate` в Preset (или вычислять динамически)
3. Обновлять рейтинги пресетов при создании/обновлении отзыва
4. Рейтинг филамента = средний рейтинг всех отзывов

   - Если да → Вариант 1 или 4
   - Если нет → Вариант 2 или 3

2. **Нужна ли отдельная система оценки пресетов (не через отзывы)?**
   - Если да → Вариант 3
   - Если нет → Вариант 1 или 4

3. **Как вычислять рейтинг филамента, если отзывы привязаны к пресетам?**
   - Простое среднее всех отзывов
   - Взвешенное среднее по пресетам (с учетом usage_count)
