# Сравнение стандартного импорта пресетов в OrcaSlicer с нашим подходом

## Стандартный способ импорта (PresetBundle::import_json_presets)

### Как работает стандартный импорт:

1. **Метод:** `PresetBundle::import_json_presets()`
   - Расположение: `src/libslic3r/PresetBundle.cpp:1042`
   - Сигнатура:
     ```cpp
     bool import_json_presets(
         PresetsConfigSubstitutions &substitutions,
         std::string &file,
         std::function<int(std::string const &)> override_confirm,
         ForwardCompatibilitySubstitutionRule rule,
         int &overwrite,
         std::vector<std::string> &result
     )
     ```

2. **Процесс импорта:**
   - Загружает JSON из файла через `config.load_from_json()`
   - Определяет тип пресета по наличию полей:
     - `printer_settings_id` → printer preset
     - `print_settings_id` → print preset
     - `filament_settings_id` → filament preset
   - Проверяет существование пресета с таким именем
   - Если пресет системный или default, не перезаписывает (возвращает false)
   - Ищет родительский пресет (inherits)
   - Если родительский пресет не найден, использует default preset для типа
   - Создает новый пресет через `collection->load_preset()`
   - Сохраняет пресет через `preset.save()`
   - Возвращает `true` даже при ошибках (ошибки логируются)

3. **Важные особенности:**
   - **Возвращает `true` даже при ошибках** - нужно проверять `import_result` или `substitutions`
   - Если `import_result` не пустой, значит импорт успешен
   - Пресет сохраняется автоматически внутри метода
   - После импорта нужно вызвать `wxGetApp().load_current_presets()` для обновления UI

### Где используется стандартный импорт:

- `Plater.cpp` - импорт через диалог выбора файла
- `MainFrame.cpp` - импорт через меню File → Import
- `GUI_App.cpp` - импорт при запуске приложения
- `CreatePresetsDialog.cpp` - создание новых пресетов
