# Как хранится JSON в PostgreSQL

## Тип данных

В миграции используется `postgresql.JSON` (тип `json` в PostgreSQL).

## Структура хранения

**PostgreSQL JSON тип** хранится **НЕ как обычная строка**, а как специальный тип данных:

1. **JSON (тип `json`)**:
   - Хранится в текстовом формате
   - Валидируется при вставке
   - Сохраняет точный формат (пробелы, порядок ключей)
   - Пример: `{"key": "value"}`

2. **JSONB (тип `jsonb`)**:
   - Хранится в бинарном формате (более эффективно)
   - Автоматически удаляет пробелы и нормализует
   - Поддерживает индексы для быстрого поиска
   - Пример: тот же JSON, но сжатый в бинарном виде

## В FilamentHub

**Текущая реализация:**
- Используется тип `json` (не `jsonb`)
- Хранится как структурированный JSON объект
- При сохранении через SQLAlchemy: автоматически сериализуется в JSON строку
- При чтении: автоматически десериализуется в Python `dict`

**Пример в БД:**
```sql
SELECT orcaslicer_settings FROM presets WHERE id = 1;
-- Результат (текстовый JSON):
-- {"nozzle_temperature_range_low": ["190"], "filament_max_volumetric_speed": ["15"]}
```

**В Python/SQLAlchemy:**
```python
preset.orcaslicer_settings = {
    "nozzle_temperature_range_low": ["190"],
    "filament_max_volumetric_speed": ["15"]
}
# SQLAlchemy автоматически конвертирует в JSON строку при сохранении
# PostgreSQL валидирует и хранит как JSON тип
# При чтении автоматически десериализуется обратно в dict
```

## Рекомендация

**Текущий тип `json` достаточен для MVP**, но если понадобится:
- Поиск по полям внутри JSON
- Индексы для быстрого доступа
- Оптимизация производительности

То можно перейти на `JSONB` (более эффективный тип).

## Итого

**JSON хранится как специальный тип PostgreSQL**, не как простая строка. Это означает:
- ✅ Валидация при вставке
- ✅ Структурированный доступ
- ✅ Автоматическая сериализация/десериализация через SQLAlchemy
- ✅ Возможность запросов через SQL (`->`, `->>` операторы)
