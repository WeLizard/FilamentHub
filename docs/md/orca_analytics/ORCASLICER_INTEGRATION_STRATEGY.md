# Стратегия интеграции OrcaSlicer параметров в FilamentHub

## Проблема

В OrcaSlicer профиль филамента содержит **113 параметров**, а в FilamentHub сейчас только **~12 базовых** параметров в модели `Preset`.

## Варианты решения

### Вариант 1: Базовые параметры + JSON поле для расширенных

**Подход:**
- Базовые параметры (текущие 12) остаются как колонки в БД
- Новое поле `settings_json: JSONB` для хранения всех остальных параметров OrcaSlicer
- При экспорте объединяем базовые + JSON

**Плюсы:**
- ✅ Гибкость - можно хранить любые параметры
- ✅ Не нужно менять миграции для базовых параметров
- ✅ Можно хранить параметры от разных версий OrcaSlicer
- ✅ Простота - базовые параметры остаются доступными для поиска/фильтрации

**Минусы:**
- ⚠️ JSON поля сложнее индексировать
- ⚠️ Нужно валидировать JSON структуру

**Пример структуры:**
```sql
CREATE TABLE presets (
    -- Текущие колонки
    id SERIAL PRIMARY KEY,
    extruder_temp FLOAT,
    bed_temp FLOAT,
    -- ...
    
    -- Новое поле
    settings_json JSONB  -- { "pressure_advance": "0.038", "adaptive_pressure_advance": "1", ... }
);
```

### Вариант 2: Только базовые параметры, остальное подставляем при экспорте

**Подход:**
- Храним только текущие базовые параметры в БД
- При экспорте в OrcaSlicer подставляем значения по умолчанию из базового профиля OrcaSlicer
- Пользователь не может настроить расширенные параметры в FilamentHub

**Плюсы:**
- ✅ Простота - не нужно менять БД/UI
- ✅ Быстрая реализация

**Минусы:**
- ❌ Нельзя настроить расширенные параметры (pressure advance, adaptive PA, и т.д.)
- ❌ Меньше гибкости для продвинутых пользователей
- ❌ Профили менее точные

### Вариант 3: JSON поле для всех параметров + базовые как кеш

**Подход:**
- Все параметры OrcaSlicer хранятся в `settings_json: JSONB`
- Базовые параметры - это вычисляемые свойства (getters) или кеш из JSON
- При сохранении базовых параметров обновляем JSON

**Плюсы:**
- ✅ Единая структура данных
- ✅ Все параметры доступны
- ✅ Можно легко расширять

**Минусы:**
- ⚠️ Сложнее работать с базовыми параметрами (нужно парсить JSON)
- ⚠️ Поиск/фильтрация по базовым параметрам сложнее

### Вариант 4: Гибридный подход - базовые + расширенные в JSON (РЕКОМЕНДУЕМЫЙ)

**Подход:**
1. **Базовые параметры** (текущие 12) остаются как колонки БД для:
   - Поиска и фильтрации
   - Отображения в карточках
   - Рекомендаций (weighted average)

2. **Расширенные параметры** хранятся в `settings_json: JSONB`:
   - Все 113 параметров OrcaSlicer
   - Параметры, которые пользователь настроил явно
   - Параметры по умолчанию для экспорта

3. **UI подход:**
   - Основная форма - только базовые параметры (как сейчас)
   - Кнопка "Расширенные настройки" - раскрывает все параметры
   - Режим "Профессионал" - показывает все 113 параметров

**Плюсы:**
- ✅ Лучшее из обоих миров
- ✅ Новым пользователям не нужно настраивать 113 параметров
- ✅ Продвинутым пользователям доступны все настройки
- ✅ Быстрый поиск по базовым параметрам
- ✅ Полная совместимость с OrcaSlicer

**Минусы:**
- ⚠️ Нужно поддерживать синхронизацию базовых ↔ JSON
- ⚠️ Нужно создать UI для расширенных настроек

## Рекомендация: Вариант 4 (Гибридный)

### Структура БД:

```python
class Preset(Base):
    # Базовые параметры (как сейчас)
    extruder_temp: float
    bed_temp: float
    print_speed: float
    # ...
    
    # Новое поле для расширенных параметров
    orcaslicer_settings: Mapped[dict | None] = mapped_column(
        JSON, nullable=True
    )  # Все 113 параметров OrcaSlicer
    
    # Синхронизация при сохранении
    def sync_basic_to_orcaslicer(self):
        """Синхронизировать базовые параметры в JSON."""
        if not self.orcaslicer_settings:
            self.orcaslicer_settings = {}
        
        self.orcaslicer_settings.update({
            "nozzle_temperature": [str(int(self.extruder_temp))],
            "bed_temp": [str(int(self.bed_temp))],
            # ...
        })
```

### UI подход:

1. **Основная форма создания/редактирования:**
   - Текущие базовые поля
   - Кнопка "Расширенные настройки OrcaSlicer" (collapsible)

2. **Расширенные настройки:**
   - Группировка по категориям:
     - Температуры
     - Вентиляторы
     - Retraction
     - Pressure Advance
     - Мультитул
     - G-code
   - Кнопка "Загрузить из профиля OrcaSlicer" (JSON файл)
   - Кнопка "Экспортировать в OrcaSlicer" (JSON файл)

3. **Отображение:**
   - В карточках пресетов - только базовые параметры
   - На странице детального просмотра - все параметры с возможностью редактирования

### Миграция:

```python
# 1. Добавить поле settings_json
# 2. При экспорте заполнять из базовых параметров + JSON
# 3. При импорте из OrcaSlicer - парсить JSON и заполнять базовые + расширенные
```
