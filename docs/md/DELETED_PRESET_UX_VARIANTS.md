# Варианты реализации обработки удалённых пресетов

## Анализ сценариев использования

### Сценарии:
1. **Случайное удаление** - пользователь случайно удалил пресет, хочет восстановить
2. **Намеренное удаление** - пользователь намеренно удалил пресет, не нужен
3. **Удаление только локально** - пользователь удалил пресет локально, но хочет оставить в FilamentHub
4. **Удаление везде** - пользователь удалил пресет локально и хочет удалить из FilamentHub
5. **Множественные удаления** - пользователь удалил несколько пресетов

---

## Вариант 1: Умное восстановление с минимальными диалогами ⭐ РЕКОМЕНДУЮ

### Подход:
- **По умолчанию**: Автоматически восстанавливать все удалённые пресеты (без диалогов)
- **При первом обнаружении**: Показать одно уведомление с опциями:
  - "Восстановить все удалённые пресеты" (по умолчанию)
  - "Удалить все из FilamentHub"
  - "Настроить поведение..."
- **Настройка в UI**: В настройках FilamentHub можно выбрать поведение для будущих случаев
- **Дополнительно**: В диалоге синхронизации показывать уведомление о восстановленных пресетах

### Плюсы:
✅ Минимум диалогов - не раздражает пользователя
✅ Безопасность - пресеты не теряются случайно
✅ Гибкость - можно настроить поведение
✅ Удобно для новичков - простое восстановление

### Минусы:
❌ Не понятно намерение пользователя (случайно удалил или намеренно)

---

## Вариант 2: Групповой диалог с умным определением

### Подход:
- **Собирать все удалённые пресеты** перед синхронизацией
- **Показать один диалог** со списком всех удалённых пресетов:
  ```
  Обнаружено 3 удалённых пресета:
  - PLA Standard [FilamentHub]
  - PETG High Temp [FilamentHub]
  - ABS Professional [FilamentHub]
  
  Что сделать?
  [ ] Восстановить все в OrcaSlicer (по умолчанию)
  [ ] Удалить все из FilamentHub
  [ ] Показать список для выбора каждого
  
  [Применить ко всем] [Настроить поведение...] [Отмена]
  ```
- **Чекбокс "Больше не спрашивать"** → сохраняет выбор в настройки
- **Если выбрано "Показать список"** → отдельный диалог для каждого пресета

### Плюсы:
✅ Один диалог для множественных пресетов
✅ Видно все удалённые пресеты
✅ Можно выбрать действие для каждого
✅ Опция "не спрашивать" для удобства

### Минусы:
❌ Сложнее в реализации
❌ Если пресетов много - диалог будет большим

---

## Вариант 3: Тихая синхронизация с настройками

### Подход:
- **По умолчанию**: Автоматически восстанавливать все удалённые пресеты (без диалогов)
- **Настройка в UI FilamentHub**:
  ```
  Поведение при удалении пресетов:
  ○ Автоматически восстанавливать (рекомендуется)
  ○ Удалять из FilamentHub
  ○ Спрашивать каждый раз
  ○ Пропускать (ничего не делать)
  ```
- **В диалоге синхронизации** показывать уведомление:
  ```
  "Синхронизация завершена. Восстановлено 3 пресета."
  ```
- **Кнопка "Отменить восстановление"** в уведомлении

### Плюсы:
✅ Минимум прерываний - не мешает работе
✅ Простая настройка - можно выбрать поведение
✅ Можно отменить восстановление - гибкость
✅ Удобно для продвинутых пользователей

### Минусы:
❌ Может быть неочевидно для новичков
❌ Может восстанавливать нежелательные пресеты

---

## Вариант 4: Гибридный подход ⭐⭐ ЛУЧШИЙ

### Подход:
- **Первое обнаружение**: Показать информативный диалог:
  ```
  Обнаружен удалённый пресет "PLA Standard [FilamentHub]"
  
  Что сделать?
  [Восстановить] [Удалить из FilamentHub] [Пропустить]
  
  ☑ Применить ко всем удалённым пресетам (3 шт.)
  ☑ Запомнить выбор и больше не спрашивать
  ```
- **Чекбокс "Запомнить выбор"** → сохраняет в настройки
- **Чекбокс "Применить ко всем"** → применяет к остальным без дополнительных диалогов
- **Настройка в UI**: Можно изменить поведение позже
- **Если выбрано "Запомнить"** → в будущем работает автоматически

### Плюсы:
✅ Баланс контроля и автоматизации
✅ Можно выбрать для всех сразу
✅ Можно запомнить выбор
✅ Настройка доступна в UI

### Минусы:
❌ Средняя сложность реализации

---

## Рекомендация: Вариант 4 (Гибридный) или Вариант 1 (Умное восстановление)

### Детали реализации Гибридного подхода:

#### 1. Первый запуск после удаления:
- Показать диалог с информацией об удалённом пресете
- Кнопки: "Восстановить" (по умолчанию), "Удалить из FilamentHub", "Пропустить"
- Чекбоксы: "Применить ко всем (N)" и "Запомнить выбор"

#### 2. Настройка в UI:
- В настройках FilamentHub раздел "Поведение при удалении пресетов"
- Радио-кнопки:
  - "Восстанавливать автоматически"
  - "Удалять из FilamentHub"
  - "Спрашивать каждый раз"
  - "Пропускать"
- По умолчанию: "Спрашивать первый раз, затем восстанавливать"

#### 3. Логика работы:
```
Если настройка = "ask" (по умолчанию):
  - Первый раз: показать диалог
  - Если выбрано "Запомнить": сохранить выбор в настройки
  - Если выбрано "Применить ко всем": применить к остальным без диалогов

Если настройка = "import":
  - Автоматически восстанавливать все удалённые пресеты

Если настройка = "delete":
  - Автоматически удалять все удалённые пресеты из FilamentHub

Если настройка = "skip":
  - Пропускать все удалённые пресеты (ничего не делать)
```

#### 4. Уведомления:
- После синхронизации: "Синхронизация завершена. Восстановлено 3 пресета."
- Или: "Синхронизация завершена. Удалено 2 пресета из FilamentHub."
