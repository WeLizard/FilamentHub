### 1. Филаменты из OrcaSlicer без производителя
- **Было:** Создавались филаменты с `brand_id="User Materials"`
- **Стало:** Создаются филаменты с `brand_id=None` (без производителя)
- **Файлы:** `orca_sync.py` (строки 1552, 1572, 1591, 1760)

- **Было:** `sync_enabled` автоматически устанавливался в `True` при создании/обновлении пресетов
- **Стало:** `sync_enabled` меняется ТОЛЬКО пользователем в UI
- **Файлы:** `orca_sync.py` (строки 2027, 2096, 2400)

### 3. Проверка sync_enabled при импорте из OrcaSlicer
- **Было:** Проверялся `sync_enabled` при импорте из OrcaSlicer (неправильно)
- **Стало:** Проверка убрана - при импорте пресет всегда обрабатывается
- **Файлы:** `orca_sync.py` (строки 1631-1667, 1690-1722)

### 4. Зацикливание при пустом списке пресетов
- **Было:** Рекурсивный вызов `synchronize_presets(true)` без защиты
- **Стало:** Добавлен флаг `m_full_sync_attempted` для защиты от зацикливания
- **Файлы:** `FilamentHubPanel.cpp` (строки 1190-1230), `FilamentHubPanel.hpp` (строка 563)

### 5. Обновление last_sync_time до завершения импорта
- **Было:** `last_sync_time` обновлялся до импорта всех пресетов из очереди
- **Стало:** `last_sync_time` обновляется ПОСЛЕ успешного импорта всех пресетов
- **Файлы:** `FilamentHubPanel.cpp` (строки 1413-1420, 2803-2820), `FilamentHubPanel.hpp` (строка 576)

### 6. Экспорт черновиков
- **Было:** Черновики (active=False) экспортировались

## 🔄 Логика синхронизации

### Первая синхронизация (force_full_sync=true):
1. Загружается токен из AppConfig
2. Валидируется токен через API
3. `updated_since=""` (пустая строка)
4. API возвращает ВСЕ пресеты с `sync_enabled=True`
5. Пресеты импортируются через очередь
6. После завершения импорта обновляется `last_sync_time`

### Последующие синхронизации (force_full_sync=false):
1. Загружается токен из AppConfig
2. Валидируется токен через API
3. Загружается `last_sync_time` из AppConfig
4. API возвращает только пресеты, обновленные после `last_sync_time` с `sync_enabled=True`
5. Пресеты импортируются через очередь
6. После завершения импорта обновляется `last_sync_time`

### Защита от зацикливания:
- Если API возвращает пустой список при инкрементальной синхронизации
- И `last_sync_time` существует
- И `m_full_sync_attempted=false`
- → Очищается `last_sync_time`, устанавливается `m_full_sync_attempted=true`, запускается полная синхронизация
- Если при полной синхронизации список снова пуст → синхронизация завершается без зацикливания

## 🎯 Ключевые принципы

1. **sync_enabled контролирует ТОЛЬКО экспорт из FilamentHub в OrcaSlicer**
   - При импорте из OrcaSlicer пресет всегда обрабатывается
   - При экспорте из FilamentHub проверяется `sync_enabled=True`

2. **Филаменты из OrcaSlicer БЕЗ производителя**
   - `brand_id=None` для всех черновиков из OrcaSlicer
   - Не используется "User Materials" бренд

3. **last_sync_time обновляется ПОСЛЕ импорта**
   - Предотвращает потерю пресетов при прерывании синхронизации

4. **Защита от зацикливания**
   - Флаг `m_full_sync_attempted` предотвращает бесконечные рекурсивные вызовы
