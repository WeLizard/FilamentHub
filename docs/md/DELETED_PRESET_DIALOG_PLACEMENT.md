# Где показывать диалог для удалённых пресетов?

## Текущая проблема

### Сценарий:
1. Пользователь открывает OrcaSlicer
2. Переходит на вкладку **FilamentHub** → запускается синхронизация
3. Пользователь переходит на вкладку **Prepare** (работает с моделью)
4. Синхронизация в фоне обнаруживает удалённые пресеты
5. **Диалог появляется внезапно**, хотя пользователь на другой вкладке

### Проблема:
- Диалог показывается относительно `this` (FilamentHubPanel)
- Пользователь может быть на другой вкладке
- Диалог блокирует работу с приложением
- Плохой UX - пользователь не понимает, откуда взялся диалог

---

## Варианты решения

### Вариант 1: Диалог относительно главного окна ⭐ РЕКОМЕНДУЮ

#### Подход:
- Показывать диалог относительно `wxGetApp().mainframe` или `nullptr` (центр экрана)
- Диалог всегда будет виден пользователю
- Не зависит от того, на какой вкладке пользователь

#### Реализация:
```cpp
// Вместо: this (FilamentHubPanel)
wxWindow* parent = wxGetApp().mainframe ? wxGetApp().mainframe : nullptr;

wxMessageBox(
    message,
    title,
    wxYES_NO | wxCANCEL | wxYES_DEFAULT | wxICON_QUESTION,
    parent  // Главное окно или nullptr (центр экрана)
);
```

#### Плюсы:
✅ Всегда видно пользователю
✅ Не зависит от активной вкладки
✅ Стандартное поведение для приложения

#### Минусы:
❌ Все равно блокирующий диалог
❌ Может появиться, когда пользователь не на вкладке FilamentHub

---

### Вариант 2: Отложенный диалог после синхронизации ⭐⭐ ЛУЧШИЙ

#### Подход:
- **Собирать все удалённые пресеты** во время синхронизации (не показывать диалоги сразу)
- **После завершения синхронизации** показать один диалог со списком всех удалённых пресетов
- **Показывать только если** пользователь на вкладке FilamentHub
- **Если на другой вкладке** → показать при следующем открытии вкладки FilamentHub

#### Реализация:
```cpp
// Во время синхронизации - собираем удалённые пресеты
struct DeletedPreset {
    int preset_id;
    std::string preset_name;
};

std::vector<DeletedPreset> deleted_presets;

// После синхронизации - показываем диалог
if (!deleted_presets.empty() && this->IsShown()) {
    // Показываем диалог только если вкладка FilamentHub видна
    show_deleted_presets_dialog(deleted_presets);
} else if (!deleted_presets.empty()) {
    // Сохраняем для показа при следующем открытии вкладки
    save_pending_deleted_presets(deleted_presets);
}

// В Show() или init() - проверяем pending deleted presets
if (has_pending_deleted_presets()) {
    show_deleted_presets_dialog(get_pending_deleted_presets());
}
```

#### Плюсы:
✅ Один диалог для всех удалённых пресетов
✅ Показывается только когда пользователь на вкладке FilamentHub
✅ Не блокирует работу на других вкладках
✅ Пользователь видит все удалённые пресеты сразу

#### Минусы:
❌ Средняя сложность реализации
❌ Нужно хранить pending deleted presets

### Вариант 3: Уведомление вместо диалога

#### Подход:
- Использовать **немодальное уведомление** (toast/status bar) вместо модального диалога
- В уведомлении кнопки действий: "Восстановить", "Удалить из FilamentHub", "Настроить..."
- Пользователь может продолжить работу, уведомление не блокирует

#### Реализация:
```cpp
// Показываем уведомление в статус-баре или через NotificationManager
show_notification_in_webview(
    "Обнаружено 3 удалённых пресета. Что сделать?",
    "warning"
);

// Или через NotificationManager (если есть)
wxGetApp().plater()->get_notification_manager()->push_notification(
    NotificationType::CustomNotification,
    "Обнаружено 3 удалённых пресета",
    "Восстановить | Удалить из FilamentHub | Настроить"
);
```

#### Плюсы:
✅ Не блокирует работу
✅ Пользователь может продолжить работу
✅ Менее навязчиво

#### Минусы:
❌ Может быть пропущено пользователем
❌ Не так заметно, как модальный диалог
❌ Сложнее реализовать кнопки действий в уведомлении

### Вариант 4: Гибридный подход (Уведомление + Диалог)

#### Подход:
- **Немодальное уведомление** в статус-баре: "Обнаружено 3 удалённых пресета"
- **Кнопка в уведомлении**: "Обработать удалённые пресеты"
- **При клике** → показать модальный диалог относительно главного окна
- **Диалог показывается** только когда пользователь нажмёт кнопку

#### Реализация:
```cpp

    // Показываем уведомление в статус-баре

        wxString::Format("Обнаружено %d удалённых пресетов", deleted_presets.size()),
        [this, deleted_presets]() {

            show_deleted_presets_dialog(deleted_presets);
```

#### Плюсы:
✅ Не блокирует работу сразу
✅ Пользователь видит уведомление
✅ Диалог показывается только когда пользователь готов
✅ Баланс между заметностью и удобством

#### Минусы:
❌ Сложнее реализовать (нужен NotificationManager или статус-бар)
❌ Пользователь может пропустить уведомление

## Рекомендация: Вариант 2 (Отложенный диалог) + Вариант 1 (Главное окно)

### Комбинированный подход:

1. **Собираем все удалённые пресеты** во время синхронизации (не показываем диалоги сразу)

2. **После завершения синхронизации**:
   - Если пользователь на вкладке FilamentHub → показать диалог относительно главного окна
   - Если на другой вкладке → сохранить для показа при следующем открытии вкладки FilamentHub

3. **Диалог показывается относительно главного окна** (`wxGetApp().mainframe` или `nullptr`)

4. **Один диалог для всех удалённых пресетов** с опциями:
   - "Восстановить все"
   - "Удалить все из FilamentHub"
   - "Показать список для выбора каждого"
   - "Запомнить выбор"

### Преимущества:
✅ Один диалог вместо множественных
✅ Показывается только когда пользователь на вкладке FilamentHub
✅ Не блокирует работу на других вкладках
✅ Всегда видно (относительно главного окна)
✅ Пользователь видит все удалённые пресеты сразу
