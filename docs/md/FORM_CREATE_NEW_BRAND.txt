ФОРМА "СОЗДАТЬ НОВЫЙ БРЕНД"
==================================================

ЗАГОЛОВОК ФОРМЫ:
-----------------
- Иконка: Factory (зелёная)
- Текст: "Создать новый бренд"

КНОПКА ЗАКРЫТИЯ:
-----------------
- Иконка: X (крестик)
- Действие: Закрытие формы и сброс ошибок

ИНФОРМАЦИОННЫЙ БЛОК: "Процесс верификации"
-------------------------------------------
Иконка: Shield (синяя)
Заголовок: "Процесс верификации"
Текст:
"После подачи заявки администратор проверит предоставленные подтверждающие документы и контактные данные (корпоративный email, сайт, официальные документы) и примет решение в течение 3–5 рабочих дней. После одобрения вы автоматически получите доступ к кабинету производителя."

ПОЛЯ ФОРМЫ:
===========

1. НАЗВАНИЕ БРЕНДА *
-------------------
Лейбл: "Название бренда *"
Тип: text input
Обязательное: Да (required)
Placeholder: "ThermPlast"
Особенности: При вводе автоматически генерируется slug (если slug пустой)

2. SLUG *
--------
Лейбл: "Slug *"
Тип: text input
Обязательное: Да (required)
Placeholder: "thermplast"
Подсказка под полем: "Уникальный идентификатор для URL (только латиница, цифры и дефисы)"

3. ОПИСАНИЕ (необязательно)
----------------------------
Лейбл: "Описание (необязательно)"
Тип: textarea (3 строки)
Обязательное: Нет
Placeholder: "Краткое описание бренда..."

4. КОНТАКТНАЯ ИНФОРМАЦИЯ
-------------------------
Заголовок раздела: "Контактная информация"

4.1. Email компании
-------------------
Лейбл: "Email компании"
Тип: email input
Обязательное: Нет
Placeholder: "info@thermplast.ru"
Особенности:
- Кнопка "Использовать мой email" (если email пользователя отличается от введённого)
- Предупреждение при использовании личной почты:
  "⚠️ Вы используете личную почту. Обязательно прикрепите подтверждающие документы."

4.2. Сайт компании
------------------
Лейбл: "Сайт компании"
Тип: url input
Обязательное: Нет
Placeholder: "https://thermplast.ru"

4.3. Соцсети бренда
-------------------
Лейбл: "Соцсети бренда"
Тип: url input + кнопка "Добавить"
Placeholder: "vk.com/thermplast; t.me/thermplast"
Особенности:
- Можно добавить несколько ссылок
- Добавленные ссылки отображаются как теги с возможностью удаления (кнопка X)
- Добавление по Enter или кнопке "Добавить"

5. ОПИСАНИЕ ПОДТВЕРЖДАЮЩИХ ДОКУМЕНТОВ *
--------------------------------------
Лейбл: "Описание подтверждающих документов * (обязательно)"
Тип: textarea (4 строки)
Обязательное: Да (required)
Placeholder: "Если вы регистрируетесь с личной почты (например, @gmail.com), постарайтесь приложить любые подтверждающие материалы: доверенность, выписку из ЕГРЮЛ/ЕГРИП (для ИП), ссылки на соцсети, маркетплейсы или фото продукции с логотипом."
Подсказка под полем:
"Если вы регистрируетесь с личной почты (не корпоративной), обязательно прикрепите подтверждающие документы ниже. Текстовое описание не заменяет документы — оно лишь поясняет их содержание."

6. ПРИКРЕПИТЬ ДОКУМЕНТЫ
-----------------------
Лейбл: "Прикрепить документы (необязательно, но ускоряют верификацию)"
Тип: file input (multiple)
Форматы: PDF, JPG, JPEG, PNG, DOC, DOCX
Максимальный размер файла: 50 MB
Текст в области загрузки:
- "Нажмите для выбора файлов"
- "PDF, JPG, PNG, DOC, DOCX (макс. 50 MB)"
Иконка: Paperclip (зелёная)
Особенности:
- Отображение списка выбранных файлов с именем и размером
- Кнопка удаления файла (X) для каждого файла

7. ДОПОЛНИТЕЛЬНОЕ СООБЩЕНИЕ
---------------------------
Лейбл: "Дополнительное сообщение (необязательно)"
Тип: textarea (2 строки)
Обязательное: Нет
Placeholder: "Любая дополнительная информация для администратора..."

ИНФОРМАЦИОННЫЙ БЛОК: "Важно"
----------------------------
Текст:
"Важно: Если вы регистрируетесь с личной почты (не корпоративной), обязательно прикрепите подтверждающие документы: доверенность от компании, письмо на фирменном бланке, выписку из ЕГРЮЛ/ЕГРИП или иные официальные материалы. При отсутствии документов заявка рассматриваться не будет."
Ссылка: "Связаться с администратором"
- Email: admin@filamenthub.ru
- Subject: "Заявка на верификацию бренда"
- Body: "Здравствуйте! Я хочу зарегистрировать бренд на платформе FilamentHub."
Иконка: Share2

ЧЕКБОКС: Подтверждение достоверности данных
-------------------------------------------
Текст:
"Я подтверждаю, что вся предоставленная информация является достоверной. Предоставление ложных сведений может повлечь отказ в верификации и блокировку аккаунта. Пользовательское соглашение"
Ссылка: "/user-agreement" (открывается в новой вкладке)
Обязательное: Да (required)
Цвет ссылки: зелёный (green-400), hover: зелёный-светлый (green-300)

КНОПКИ:
=======

1. ПОДАТЬ ЗАЯВКУ НА СОЗДАНИЕ БРЕНДА
------------------------------------
Тип: submit button
Стиль: Градиент зелёный (green-600 to emerald-600)
Текст:
- В процессе: "Отправка заявки..." (с иконкой загрузки Loader2)
- Обычное состояние: "Подать заявку на создание бренда" (с иконкой Plus)
Disabled: при загрузке или если не отмечен чекбокс подтверждения

2. ОТМЕНА
---------
Тип: button (не submit)
Стиль: Полупрозрачный белый (white/10)
Текст: "Отмена"
Действие: Закрытие формы и очистка всех полей
Disabled: при загрузке

ВАЛИДАЦИЯ:
==========

Обязательные поля:
- Название бренда *
- Slug *
- Описание подтверждающих документов *
- Чекбокс подтверждения достоверности данных *

Условия:
- Если email личный (не корпоративный), файлы обязательны
- При отсутствии документов при личной почте - заявка рассматриваться не будет

ОБРАБОТКА ОШИБОК:
================
Блок ошибки:
- Фон: красный полупрозрачный (red-500/20)
- Граница: красная (red-500/50)
- Текст: красный-светлый (red-300)
- Позиция: вверху формы (после заголовка)

СТИЛИЗАЦИЯ:
===========
- Фокус на полях: зелёное кольцо (ring-green-500)
- Фон полей: полупрозрачный белый (white/10)
- Границы: полупрозрачный белый (white/20)
- Placeholder: серый (gray-400)
- Текст: белый
- Лейблы: серый-светлый (gray-300)
