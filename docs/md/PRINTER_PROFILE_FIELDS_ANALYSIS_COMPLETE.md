# Полный анализ полей PrinterProfile

## Проблемы, найденные при анализе

### 1. `printable_area` - несоответствие форматов

**Проблема:**
- **Frontend** использует формат: `{x: number, y: number}` (ширина и глубина)
- **Экспортер** ожидает формат: `{x_min, y_min, x_max, y_max}` (координаты углов)
- **OrcaSlicer** использует формат: `["0x0", "220x0", "220x220", "0x220"]` (массив строк с координатами углов)

**Текущее состояние:**
- При импорте из OrcaSlicer `printable_area` сохраняется как массив строк без преобразования
- При экспорте экспортер пытается использовать `x_min`, `y_min`, `x_max`, `y_max`, которых нет в формате frontend
- Frontend отправляет `{x, y}`, но экспортер не может с этим работать

**Решение:**
1. Добавить функцию преобразования `printable_area` из формата OrcaSlicer в наш формат
2. Унифицировать формат: использовать `{x: number, y: number}` везде (ширина и глубина)
3. При экспорте преобразовывать `{x, y}` в формат OrcaSlicer `["0x0", "Xx0", "XxY", "0xY"]`
