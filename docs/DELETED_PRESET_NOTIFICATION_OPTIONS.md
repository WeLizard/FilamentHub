# Варианты показа уведомлений об удалённых пресетах

## Анализ NotificationManager

### Возможности:
1. **Hypertext + Callback**: Можно показать уведомление с кликабельным текстом (например, "Обработать"), который вызывает callback
2. **Уровни важности**: `ErrorNotificationLevel`, `WarningNotificationLevel`, `RegularNotificationLevel`, `ImportantNotificationLevel`
3. **Автоматическое исчезновение**: Регулярные уведомления исчезают через 10 секунд, Error/Warning - не исчезают
4. **Рендеринг**: Уведомления показываются в 3D view (GLCanvas3D), в правом нижнем углу
5. **Callback возвращает bool**: `true` - уведомление закрывается, `false` - остается открытым

### Примеры использования в OrcaSlicer:
- `PresetUpdateAvailable` - показывает уведомление с hypertext "Details", при клике открывает диалог обновления пресетов
- `BBLPrinterConfigUpdateAvailable` - аналогично для конфигурации принтера
- `ExportFinished` - уведомление об успешном экспорте

## Варианты решения

### Вариант 1: NotificationManager с hypertext + callback ⭐ РЕКОМЕНДУЮ

#### Подход:
- Показываем уведомление через `NotificationManager::push_notification`
- Используем `hypertext` = "Обработать" или "Настроить"
- При клике на hypertext вызывается callback, который показывает диалог

#### Реализация:
```cpp
// После синхронизации, если есть удалённые пресеты
if (!deleted_presets.empty()) {
    std::string message = wxString::Format("Обнаружено %d удалённых пресетов", deleted_presets.size()).ToUTF8().data();
    wxGetApp().plater()->get_notification_manager()->push_notification(
        NotificationType::CustomNotification,
        NotificationManager::NotificationLevel::WarningNotificationLevel,
        message,
        _u8L("Обработать"),  // hypertext
        [this, deleted_presets](wxEvtHandler* evt_handler) {
            // Callback: показываем диалог обработки удалённых пресетов
            CallAfter([this, deleted_presets]() {
                show_deleted_presets_dialog(deleted_presets);
            });
            return true;  // Закрыть уведомление после клика
        }
    );
}
```

#### Плюсы:
✅ Использует стандартный механизм OrcaSlicer
✅ Не блокирует работу (немодальное уведомление)
✅ Пользователь может продолжить работу, уведомление висит в углу
✅ При клике открывается диалог для обработки
✅ Уведомление видно в любой момент (не зависит от вкладки)
✅ Автоматически исчезает через некоторое время (если Warning) или остается (если Error)
✅ Можно закрыть вручную (кнопка X)

#### Минусы:
❌ Пользователь может пропустить уведомление (если не смотрит в 3D view)
❌ Уведомление показывается в 3D view, а не в FilamentHub панели
❌ Нужно хранить список удалённых пресетов до клика на hypertext

### Вариант 2: Модальный диалог сразу (wxMessageBox)

#### Подход:
- Показываем модальный диалог сразу после синхронизации
- Диалог блокирует работу до выбора пользователя

#### Реализация:
```cpp
// После синхронизации, если есть удалённые пресеты
if (!deleted_presets.empty()) {
    wxWindow* parent = wxGetApp().mainframe ? wxGetApp().mainframe : nullptr;
    int result = wxMessageBox(
        wxString::Format("Обнаружено %d удалённых пресетов. Что сделать?", deleted_presets.size()),
        _L("FilamentHub: Удалённые пресеты"),
        wxYES_NO | wxCANCEL | wxYES_DEFAULT | wxICON_QUESTION,
        parent
    );
    // Обработка результата...
}
```

#### Плюсы:
✅ Гарантированно видно пользователю
✅ Немедленная реакция
✅ Простая реализация

#### Минусы:
❌ Блокирует работу (модальный диалог)
❌ Может появиться, когда пользователь не на вкладке FilamentHub
❌ Навязчиво (прерывает работу)
❌ Неудобно, если пользователь на другой вкладке

### Вариант 3: Отложенный диалог после синхронизации

#### Подход:
- Собираем все удалённые пресеты во время синхронизации
- После завершения синхронизации показываем диалог
- **Показываем только если** пользователь на вкладке FilamentHub
- **Если на другой вкладке** → сохраняем для показа при следующем открытии вкладки FilamentHub

#### Реализация:
```cpp
// Во время синхронизации - собираем удалённые пресеты
std::vector<DeletedPreset> deleted_presets;

// После синхронизации - показываем диалог
if (!deleted_presets.empty() && this->IsShown()) {
    // Показываем диалог только если вкладка FilamentHub видна
    show_deleted_presets_dialog(deleted_presets);
} else if (!deleted_presets.empty()) {
    // Сохраняем для показа при следующем открытии вкладки
    save_pending_deleted_presets(deleted_presets);
}

// В Show() или init() - проверяем pending deleted presets
if (has_pending_deleted_presets()) {
    show_deleted_presets_dialog(get_pending_deleted_presets());
}
```

#### Плюсы:
✅ Один диалог для всех удалённых пресетов
✅ Показывается только когда пользователь на вкладке FilamentHub
✅ Не блокирует работу на других вкладках
✅ Пользователь видит все удалённые пресеты сразу

#### Минусы:
❌ Средняя сложность реализации (нужно хранить pending deleted presets)
❌ Все равно блокирующий диалог (хотя и в подходящий момент)
❌ Нужно проверять, видна ли вкладка FilamentHub

### Вариант 4: Комбинация: NotificationManager + Отложенный диалог ⭐⭐ ЛУЧШИЙ

#### Подход:
1. **Собираем все удалённые пресеты** во время синхронизации (не показываем диалоги сразу)
2. **После завершения синхронизации**:
   - Показываем уведомление через `NotificationManager` с hypertext "Обработать"
   - Уведомление висит в углу 3D view
3. **При клике на hypertext**:
   - Показываем модальный диалог относительно главного окна
   - Диалог показывает список всех удалённых пресетов
   - Пользователь может выбрать действие для каждого пресета

#### Реализация:
```cpp
// После синхронизации, если есть удалённые пресеты
if (!deleted_presets.empty()) {
    // Сохраняем список удалённых пресетов для диалога
    m_pending_deleted_presets = deleted_presets;
    
    // Показываем уведомление
    std::string message = wxString::Format("Обнаружено %d удалённых пресетов", deleted_presets.size()).ToUTF8().data();
    wxGetApp().plater()->get_notification_manager()->push_notification(
        NotificationType::CustomNotification,
        NotificationManager::NotificationLevel::WarningNotificationLevel,  // Не исчезает автоматически
        message,
        _u8L("Обработать"),  // hypertext
        [this](wxEvtHandler* evt_handler) {
            // Callback: показываем диалог обработки удалённых пресетов
            CallAfter([this]() {
                if (!m_pending_deleted_presets.empty()) {
                    show_deleted_presets_dialog(m_pending_deleted_presets);
                    m_pending_deleted_presets.clear();  // Очищаем после показа
                }
            });
            return true;  // Закрыть уведомление после клика
        }
    );
}

void FilamentHubPanel::show_deleted_presets_dialog(const std::vector<DeletedPreset>& presets) {
    wxWindow* parent = wxGetApp().mainframe ? wxGetApp().mainframe : nullptr;
    // ... показываем диалог с списком пресетов
}
```

#### Плюсы:
✅ Не блокирует работу сразу (немодальное уведомление)
✅ Пользователь видит уведомление в углу 3D view
✅ При клике открывается диалог для обработки
✅ Уведомление не исчезает автоматически (Warning level)
✅ Диалог показывается только когда пользователь готов (кликнул на hypertext)
✅ Использует стандартный механизм OrcaSlicer
✅ Баланс между заметностью и удобством

#### Минусы:
❌ Пользователь может пропустить уведомление (если не смотрит в 3D view)
❌ Нужно хранить список удалённых пресетов до клика на hypertext
❌ Средняя сложность реализации

## Сравнение вариантов

| Вариант | Блокирует работу | Заметность | Сложность | Стандартность |
|---------|------------------|------------|-----------|---------------|
| 1. NotificationManager | ❌ Нет | ⚠️ Средняя | ✅ Низкая | ✅ Да |
| 2. Модальный диалог | ✅ Да | ✅ Высокая | ✅ Низкая | ⚠️ Частично |
| 3. Отложенный диалог | ⚠️ Частично | ✅ Высокая | ⚠️ Средняя | ⚠️ Частично |
| 4. NotificationManager + Диалог | ❌ Нет | ⚠️ Средняя | ⚠️ Средняя | ✅ Да |
| 5. Статус-бар | ❌ Нет | ✅ Высокая | ❓ Неизвестно | ❓ Неизвестно |

## Рекомендация: Вариант 4 (NotificationManager + Отложенный диалог)

### Почему:
1. **Не блокирует работу**: Пользователь может продолжить работу, уведомление висит в углу
2. **Заметность**: Уведомление Warning level не исчезает автоматически, висит до клика
3. **Стандартность**: Использует стандартный механизм OrcaSlicer (как PresetUpdateAvailable)
4. **Удобство**: Диалог показывается только когда пользователь готов (кликнул на hypertext)
5. **Гибкость**: Можно добавить несколько действий в диалоге (восстановить все, удалить все, показать список)

### Альтернатива: Вариант 1 (только NotificationManager)

Если не хотим показывать диалог, можно использовать только уведомление:
- Показываем уведомление с hypertext "Восстановить" или "Удалить из FilamentHub"
- При клике на hypertext выполняем действие (восстановить или удалить)
- Можно показать несколько уведомлений (по одному на пресет) или одно уведомление с общим действием

## Вопросы для обсуждения:

1. **Какой вариант предпочтительнее?**
   - Вариант 4 (NotificationManager + Диалог) - рекомендую
   - Вариант 1 (только NotificationManager) - проще
   - Вариант 3 (Отложенный диалог) - без уведомления

2. **Что делать, если пользователь не кликнул на уведомление?**
   - Оставить уведомление висеть (Warning level не исчезает)
   - Показать диалог при следующем открытии вкладки FilamentHub
   - Сохранить pending deleted presets и показать при следующей синхронизации

3. **Какие действия должны быть в диалоге?**
   - "Восстановить все" - импортировать все удалённые пресеты обратно
   - "Удалить все из FilamentHub" - удалить все удалённые пресеты из FilamentHub
   - "Показать список" - показать список пресетов с выбором действия для каждого
   - "Запомнить выбор" - запомнить выбор пользователя и применять автоматически в будущем

4. **Нужно ли показывать диалог, если пользователь на другой вкладке?**
   - Да, показать сразу (Вариант 2)
   - Нет, показать при следующем открытии вкладки FilamentHub (Вариант 3)
   - Нет, показать уведомление, диалог при клике (Вариант 4)
